=== Get My Weekly Duty Pattern

operation::duty-pattern/get-mine[]

[NOTE]
====
The saved weekday pattern is independent of duty-type visibility. Automatic
application is available only while the team has exactly one visible duty
type. With zero or multiple visible types, the pattern remains saved and duty
responses use `PATTERN_PAUSED`; automatic application resumes with the single
visible type when the ambiguity is removed.

Calendar lookup materializes missing pattern dates as `Duty` rows without
overwriting existing dates. Pattern update or deletion takes effect from today
in Asia/Seoul and deletes every duty from today onward, including manual
entries, before the new pattern is materialized on the next lookup.
====

=== Update My Weekly Duty Pattern

operation::duty-pattern/update-mine[]

=== Delete My Weekly Duty Pattern

operation::duty-pattern/delete-mine[]

[NOTE]
====
Duty and weekly-pattern mutation endpoints can return HTTP `409` with code
`common.concurrentUpdate` when another update holds the member lock or the
bounded transaction cannot finish in time. Clients should refresh the current
state before retrying.
====
